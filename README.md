## 🚀 使用说明

如果您需要参考这个库学习笔者当时的学习思路，请按照如下方式部署：

1. 克隆仓库：

如果您使用的是 HTTPS 协议克隆的话，请使用

```powershell
git clone https://github.com/RainstormX5702005/2026.1.30-MCM-Resources-Index.git
```

2. 安装相关依赖：

为了便于代码的管理，建议您使用 **uv** 作为 python 环境的管理工具，在 clone 以后输入

```powershell
cd your\project\directory
uv sync
```

之后，您就需要使用 **uv** 即可在终端执行代码，比如您可以这么运行代码，更多指令请查阅相关文档：

```powershell
uv run .\path\to\your\file.py
```

注意：

- 请确保同时提交 `pyproject.toml` 与 `uv.lock` 
- 请不要把 `.venv` 等本机环境相关的文件上传到本库，否则本库会拒绝您的提交

本人在此介绍各个文件夹的功能，您在提交代码的时候务必把代码装入相应的文件夹中，否则此次提交会被拒绝：

- docs: 用于存放与比赛相关的文档，可以存储任意与比赛模型或是代码相关的内容
- src: 用于存放本次 2026MCM 的相关代码
    - template: 用于存放预先编译好的相关模型或是数据可视化代码片段
    - assets: 用于存放最终训练出来的图片
    - configs: 用于存放代码所需的工作区配置信息
- examples: 用于存放所有算法相关的训练内容信息，请您根据已有的分类将您的训练代码存放进去，或是建立新的文件夹避免混淆

其余根目录文件非必要请勿删除，防止因环境破坏而导致编译失败！

## 🧠 建模策略

基于若干次 Kaggle 与真题的训练，本人提炼出部分建模心得，仅供参考：

1. 拿到数据集后先使用 `pandas` 库中的 `df.info()` 先行查看数据的行列类型与缺失值状况，如何处理数据可以通过这个步骤直观看出。
2. 之后进入 **数据预处理** 阶段，数据预处理的阶段需要执行一些无需模型即可规整数据集相关数据的操作，使得数据可以通过一些模型的算法去填充缺失值。
3. 一般而言，做完一次预处理后可以接着进行一次 **特征工程 (Feature Engineering)**，特征工程指的是分析原有数据集中的特点基于原有列分离或构造新的特征，以便模型理解与训练。
4. 重复第二步和第三步，直到数据集能提取的主要信息被榨取完停止。
5. 训练模型之前，把所有涉及离散值的特征使用编码方式转化为机器可以理解的数字，应当根据离散的特征是否具有等级来选择合适的编码方法。
5. 接下来，需要根据数据集的规模与特征的个数选择一个合理的模型来训练想要的模型，其中**调参**是一个模型是否欠拟合还是过拟合的关键，与最终结果的可信度紧密挂钩。
6. 训练完毕后，可以使用交叉验证方法来判断模型的可信度，一般使用的模型为分类模型和回归模型：
    - 分类模型：用 **混淆矩阵 (Confusion Matrix)** 与 ACU 参数来评估。
    - 回归模型：用 $R^2$, MSE, MSE_MEAN, MSE_STD 等参数来评估。
8. 接下来只需要对需要预测的模型做上述步骤的处理，并且直接调用已训练好的模型即可完成建模任务！

尽管写起来比较直观，但是其中具有很多的学问，希望读者在学习的过程中多翻阅 `scikit-learn` 的参考文档，从中可以获取到的知识远比本笔记可获得的多。

## 🧬 核心算法

本库采用 Python 的 `pandas`, `numpy`, `scikit-learn` 与部分深度学习库进行模型的构造，训练与评价。

建模的第一核心要义在于 **特征工程 (Feature Engineering)** 与 **数据预处理 (Data Preprocessing)**，为了避免 “垃圾入，垃圾出” 的尴尬情况，如何处理粗数据成为了非常重要的一个话题。

幸运的是 `scikit-learn` 提供了足够多的预处理方法，选取合适的算法处理这些数据是建模手应该关注的问题。

然而，数据的准备决定上限，模型的选取则直接决定下限，根据合适的建模情况选取合适的模型做合理的预测也是建模手与编程手应该考虑的问题。

本部分便以这两个视角展开，详细讨论这些模型的使用要义与限制。
